Scala Meets Data Engineering: Functional Tools I Wish Existed

🔁 “I got tired of repeating myself. So I wrote Scala code that repeats me better than I do.”

Growing up in data engineering, I constantly saw the same patterns: filter, map, explode, transform—not once, but dozens of times across projects. I realized I needed tools that were like helpers on speed dial. Enter the Data Engineering Functional Extensions I built and shared on GitHub.

🚀 What’s In the Repo


🔗 View on GitHub: vim89/dataengineering-savvy

Inside, you’ll find:

  • SeqOps: extension methods on Seq[T] to handle common patterns like safeHead, distinctByKey, etc.
  • DatasetUtils: reusable implicit helpers for Spark Datasets—withColumnOpt, filterIf, dropEmptyPartitions.

💡 Example: Chaining with Elegant Intent

import com.vitthal.fp.SeqOps._

val ids = Seq("a", "", "c", "a")
val clean = ids
  .safeDistinctBy(identity)
  .filterNonEmpty
  .sortBy(_.toLowerCase)

assert(clean == Seq("a", "c"))

import spark.implicits._
import com.vitthal.fp.DatasetUtils._

val df = Seq(("Alice", 20), ("Bob", null.asInstanceOf[Integer])).toDF("name", "age")

val normalized = df
  .withColumnOpt("age", _.coalesce(lit(0)))
  .filterIf("age > 0", df("age") > 0)

These helpers cut down hundreds of lines of boilerplate when building pipelines.
